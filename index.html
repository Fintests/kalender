Kalander

Prototype based calendar

Kalander (from Kala in Sanskrit meaning time) uses and extends the scrip.aculo.us library. Its aim is to ease the use of sql formatted dates in web applications without the need to use 3 inputs to do the work.

Examples

The following examples show a few of the options that you can use.

Add any javascript function to launch after you select a date:
{ onComplete:function(element){ new Effect.Highlight(element) } }

Ensure the dates are only working week days:
Result:
{ weekends:false }

Restrict dates by providing an array of sql formatted dates:
Result:
{ disabled: ['2006-12-18','2006-12-25','2006-12-31'] }

Only select dates after a given sql formatted date:
Result:
{ after: '2006-12-18' }

Only allow dates to be selected after a given date displayed in a form element:
Result:
{ after: 'date_1' }

The same format (sql or id of a form element) can be used to ensure the date is before another:
Result:
{ before: 'date_1' }
